# 文档规范（示例）

> 策划案 skill 出稿时对照本规范。这是演示样例，按团队实际情况修改。

## 命名

- 文件名（slug）：`模块-主题.md`，kebab-case，中文可，例如 `战斗-连击系统.md`
- **版本不进文件名**：版本演进由 git 历史 + 文末「变更记录」承载（与 operations.md §9 一致）
- 待定稿放 `documents/pending/`，定稿后由晋级 PR 移入 `documents/published/`

## 文档结构

每份策划案至少包含：

1. **概述**：一句话说清这份文档解决什么
2. **目标 / 非目标**：明确范围边界
3. **详细设计**：核心内容
4. **关联**：引用的知识库条目、相关文档、对应 Issue 号
5. **变更记录**：版本 + 改动摘要

## 写作约定

- 术语统一使用知识库 `knowledge/` 中的定义
- 数值 / 公式单列表格，便于后续平衡性调整
- 每份文档头部注明：作者、对应 Issue、目标里程碑
